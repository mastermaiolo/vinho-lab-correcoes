🔒 Práticas Cruciais de Segurança

Uma chave de API é como a senha do seu cofre. Se alguém tiver acesso a ela, poderá usar a cota de IA em seu nome, podendo esgotar seus limites ou gerar custos!

❌

Método Inseguro (Direto no Front-end)

Navegador do Usuário → expõe chave publicamente → Código JS do Site

Qualquer pessoa pode abrir as "Ferramentas do Desenvolvedor" (F12) no navegador e roubar sua chave.

✅

Método Seguro (Com Servidor Intermediário)

Navegador → requisição segura → Seu Servidor/Backend (Node/Python) → envia chave e recebe IA → Google Gemini API

A chave fica guardada em segurança no seu servidor backend (variáveis de ambiente) e nunca é enviada ao navegador do usuário.

⚠️

Nunca envie sua chave de API para o GitHub!

Robôs escaneiam repositórios públicos no GitHub a cada segundo em busca de chaves expostas. Adicione arquivos que contenham chaves (como arquivos .env) ao seu .gitignore.
